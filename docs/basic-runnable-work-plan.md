# 达到基本可运行状态的开发清单

## 1. 文档用途

本文件用于定义“项目达到基本可运行状态”所需的最小开发范围。

这里的“基本可运行状态”不是指：

- 后端局部接口可以返回数据
- 只有导演页能看到部分真实信息
- 需要手工拼接多个接口才能理解世界状态

这里的“基本可运行状态”指：

- 本地轻量模式可以稳定启动
- 用户打开 Web 页面后，能看到真实世界状态
- 用户能主动推进世界
- 用户至少能读取并参与一条核心社交链路
- 导演模式能继续作为辅助观察视图存在

## 2. 当前基础

当前已经具备的基础包括：

- world runtime、scheduler、event、persistence 主链路已跑通
- 群聊、私聊、朋友圈基础消息写入已跑通
- `/api/world/state`、`/api/world/advance`、social API 已可用
- 导演页已接入真实后端数据
- relationship 已具备最小持久化与行动驱动更新能力

因此，当前最缺的不是继续搭骨架，而是把这些后端能力组织成“用户可直接操作”的最小产品闭环。

## 3. 基本可运行状态定义

达到本阶段时，应满足以下条件：

1. 首页或世界主页可展示真实 world state，而不是静态说明。
2. 用户可在前端主动触发一次 world advance，并立即看到状态变化。
3. 用户可查看默认群聊，并发送一条群聊消息。
4. 用户可查看朋友圈动态流，并发布一条动态。
5. 导演页继续可用，并能读取角色、关系、事件与会话预览。
6. 本地轻量模式启动步骤清晰，前后端都能跑起来。

## 4. 推荐开发顺序

### 任务 1：世界主页真实化

当前状态：

- 已完成

内容：

- 首页接入 `/api/world/state`
- 展示 world clock、active characters、recent events、pending tasks
- 保留项目介绍，但不再只有静态文案

验收标准：

- 打开首页即可看到真实世界状态
- API 不可达时能优雅降级

### 任务 2：世界推进入口

当前状态：

- 已完成

内容：

- 在首页或世界主页增加“推进世界”入口
- 调用 `/api/world/advance`
- 推进后刷新 world state 视图

验收标准：

- 用户可在前端主动触发世界推进
- 推进后能观察到 recent events 或 pending tasks 的变化

### 任务 3：最小群聊页

当前状态：

- 已完成

内容：

- 新增群聊页
- 读取 conversations，默认进入 `conv-general`
- 展示消息列表
- 支持发送群聊消息

验收标准：

- 用户能看到真实群聊消息
- 用户能发送一条群聊消息并看到结果

### 任务 4：最小朋友圈页

当前状态：

- 已完成

内容：

- 新增朋友圈页
- 展示 `conv-moments` 对应动态流
- 支持发布一条 moment

验收标准：

- 用户能看到真实朋友圈动态
- 用户能发送一条新动态

### 任务 5：本地运行入口收束

当前状态：

- 已完成

内容：

- 检查并补齐本地开发所需的 API base URL 使用方式
- 明确首页、群聊页、朋友圈页在本地轻量模式下的默认联通方式
- 如有必要，补充 README 或脚本说明

验收标准：

- 轻量模式启动后，前端页面默认能连上本地 API
- 不需要反复手动改代码才能本地运行

## 5. 本阶段暂不强制要求

为了尽快达到“基本可运行状态”，以下内容可暂缓：

- Docker / PostgreSQL / Redis 联调
- plan 完整持久化闭环
- moment comment / like
- director 控制能力
- 真实 LLM 表达层
- WebSocket 实时推送
- 更复杂的关系规则系统

这些工作很重要，但不属于“先让产品能直接玩起来”的最短路径。

## 6. 完成后再进入的工作

本阶段完成后，建议再继续以下顺序：

1. 环境与一致性验证
2. plan 持久化与更完整领域补全
3. 导演控制与延迟可见规则
4. 表达层升级
5. 基础设施深化
